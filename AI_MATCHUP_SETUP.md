# AI Matchup Analyzer - Setup Guide

This guide explains how to set up and configure the AI Matchup Analyzer feature that integrates your Vercel frontend with your Render backend.

## Architecture

- **Frontend**: Deployed on Vercel (React/Vite)
- **Backend**: Deployed on Render (Node.js/Express)
- **AI Providers**: OpenAI (GPT-4) or Anthropic Claude

## Frontend Setup (Vercel)

### 1. Environment Variables

You need to configure the backend URL in your Vercel project:

**Option A: Using Vercel Dashboard (Recommended)**
1. Go to your Vercel project settings
2. Navigate to "Environment Variables"
3. Add the following variable:
   - **Name**: `VITE_BACKEND_URL`
   - **Value**: Your Render backend URL (e.g., `https://your-backend.onrender.com`)
   - **Environments**: Production, Preview, and Development

**Option B: Using .env.local (Local Development)**
1. Copy `frontend/.env.example` to `frontend/.env.local`
2. Update `VITE_BACKEND_URL` with your backend URL:
   ```bash
   VITE_BACKEND_URL=http://localhost:3000  # for local development
   # OR
   VITE_BACKEND_URL=https://your-backend.onrender.com  # for testing with production backend
   ```

### 2. Deploy to Vercel

```bash
cd frontend
npm install
npm run build
```

Vercel will auto-deploy when you push to your connected GitHub branch.

## Backend Setup (Render)

### 1. Environment Variables

Configure the following environment variables in your Render dashboard:

**Required for OpenAI:**
```bash
OPENAI_API_KEY=sk-proj-...  # Get from https://platform.openai.com/api-keys
OPENAI_MODEL=gpt-4o
```

**Required for Claude (optional):**
```bash
ANTHROPIC_API_KEY=sk-ant-...  # Get from https://console.anthropic.com/
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022
```

**CORS Configuration:**
```bash
FRONTEND_URL=https://your-vercel-app.vercel.app
```

**Other Required Variables:**
```bash
PORT=3000
NODE_ENV=production
MONGODB_URI=your_mongodb_connection_string
GEMINI_API_KEY=your_gemini_key  # For Kelly Criterion insights
```

### 2. API Keys Setup

#### OpenAI API Key
1. Visit https://platform.openai.com/api-keys
2. Create a new API key
3. Copy the key and add it to your Render environment variables

#### Anthropic API Key (Optional)
1. Visit https://console.anthropic.com/
2. Create a new API key
3. Copy the key and add it to your Render environment variables

### 3. Deploy to Render

```bash
cd backend
npm install
npm start
```

Render will auto-deploy when you push to your connected GitHub branch.

## API Endpoint

The AI Matchup feature uses the following endpoint:

**POST** `/api/matchup`

**Request Body:**
```json
{
  "sport": "football",        // or "basketball"
  "team1": "Kansas City Chiefs",
  "team2": "Buffalo Bills",
  "provider": "openai",       // or "claude"
  "season": "2024"           // optional
}
```

**Headers:**
```
Content-Type: application/json
x-user-id: <user-identifier>  // Auto-generated by frontend
```

**Response:**
```json
{
  "success": true,
  "provider": "openai",
  "data": "AI-generated matchup analysis...",
  "timestamp": "2024-11-03T..."
}
```

## Testing

### Local Development

1. **Backend** (Terminal 1):
   ```bash
   cd backend
   npm install
   npm start
   # Server runs on http://localhost:3000
   ```

2. **Frontend** (Terminal 2):
   ```bash
   cd frontend
   npm install
   echo "VITE_BACKEND_URL=http://localhost:3000" > .env.local
   npm run dev
   # App runs on http://localhost:5173
   ```

3. Navigate to http://localhost:5173, click the "AI Matchup" tab, and test the feature.

### Production Testing

1. Visit your Vercel frontend: `https://your-app.vercel.app`
2. Click the "AI Matchup" tab
3. Enter team names and click "Analyze Matchup"
4. The frontend will call your Render backend API

## Troubleshooting

### CORS Errors
- Ensure `FRONTEND_URL` in Render matches your Vercel domain exactly
- Check that the backend CORS configuration includes `x-user-id` in allowed headers

### 401/403 Errors
- Verify the `x-user-id` header is being sent by the frontend
- Check browser console for any header-related errors

### 500 Errors
- Ensure OpenAI or Anthropic API keys are correctly configured in Render
- Check Render logs for detailed error messages

### Rate Limiting
- Default: 100 requests per 15 minutes
- Adjust `RATE_LIMIT_MAX_REQUESTS` in Render if needed

## Cost Considerations

- **OpenAI GPT-4**: ~$0.01-0.03 per analysis
- **Anthropic Claude**: ~$0.01-0.025 per analysis
- Monitor your API usage in respective dashboards

## Security Notes

- User IDs are stored in browser localStorage
- API keys are never exposed to the frontend
- All requests are validated on the backend
- Rate limiting is enforced per IP/user
